【② CrasoStorm 実装解説編（プログラムの構造・AIの作り方の要点）】

■この資料の立ち位置
・CrasoStorm（対戦シミュレーション）を「どういう構成で動かしているか」「ユニットAI（移動/攻撃）をどこに書くか」を解説する資料。

■全体像（重要ポイント）
・ユニットは「unitファイル（各ユニットのAIスクリプト）」で行動が決まる。
・行動の基本は大きく
　(1) 移動：moveTo(...) で次に向かう座標を返す
　(2) 攻撃：attackTo(...) で攻撃対象や攻撃方法（通常/スキル）を返す
　(3) そのほか：状態（スキル使用済みか等）や距離計算などをutilsが補助
…という分担。

■ユニットAI（移動）の例（資料記載の内容を要約）
・moveTo(turn, enemies, allies, enemyCastle, allyCastle, self)
　└基本方針：最も近い敵を探し、敵がいればそこへ向かう。いなければ敵城へ向かう。
　└補助：utils.findNearest(...) を使って近い対象を求める。

■ユニットAI（攻撃）の例（資料記載の内容を要約）
・attackTo(turn, enemies, allies, enemyCastle, allyCastle, self)
　└射程内に敵がいれば攻撃する。いなければ null（攻撃しない）
　└対象選択：utils.findNearest(...) で近い敵を選ぶ
　└スキル使用：utils.hasUsedSkill(self) 等で「スキル使用済みか」を見て
　　・未使用なら { target: ..., method:"skill" }
　　・使用済みなら { target: ..., method:"normal" }
　　のように返す（資料の例）。

■ジョブ文字列の扱い（資料の例）
・ジョブ名（"soldier", "guardian", "assassin" など）に応じて処理を分ける例が載っている
・たとえば switch(job){...} のように分岐してユニットごとの戦い方を変える、という説明。

※この資料はコード断片が多く、ページ全体の「完全なソース全文」ではなく「設計意図＋代表例」を説明する内容です。

【③ CrasoStorm 実行解説編（ローカルで動かす手順）】

■目的
・配布されたCrasoStormプロジェクトをローカルで起動し、ユニットAIを差し替えて実行・観察する。

■手順（資料の流れに沿って整理）
1) プロジェクトを入手してフォルダを開く
・VSCodeでプロジェクトフォルダを開く（画面例あり）

2) src 構成の確認
・src 配下に assets / config / data / engine / render / sdk / shared / teams などがある
・ルート付近に index.html / main.js / styles.css などがある（画面例あり）

3) VSCode拡張「Live Server」を入れる
・Extensionsから「Live Server」を検索してインストール（画面例あり）

4) 自分のユニットコードを反映
・teams/east/ 配下に unit01.js ～ unit10.js がある（画面例あり）
・自作したユニットコードでこれらを置き換える（コピーして上書きする想定）

5) 実行（ブラウザで起動）
・index.html を Live Server で開く（右クリック→Open with Live Server の流れの画面例）
・ブラウザでゲーム画面が起動する（例：127.0.0.1:5500/index.html の画面）

6) ゲーム開始と確認ポイント
・「戦闘開始」ボタンでゲームスタート（以後は観戦）
・開始前にBGMも選べる（ドロップダウンで選択）
・右側にイベントログが出る（移動/スキル/回復などのログ）
・画面下部にユニット情報が出る（名前/職業/HP/座標など）
・HPが0のユニットは色が薄くなる（視覚的に戦闘不能が分かる）

7) 開発サイクルの推奨
・「ユニット作成 → 実行 → 調整」を繰り返してユニットを強くしよう、という締め。
